# 错题本显示问题排查指南

## 已修复的问题

### 1. 后端API重复数据问题 ✅

**问题描述：**
- 之前的SQL查询使用LEFT JOIN会导致同一道题出现多次（如果有多次错误答题记录）
- 例如：一道题答错3次，会返回3条相同的记录

**修复方案：**
- 使用子查询获取最近一次的错误答案
- 确保每道题只返回一条记录

**修复位置：** [app.py](file://c:/Users/j2846/Desktop/my%20attempt/flask_app/backend/app.py#L829-L871)

```python
# 修复前：LEFT JOIN 导致重复
LEFT JOIN answer_records ar ON w.question_id = ar.question_id 

# 修复后：子查询获取最近一次
(
    SELECT ar.user_answer 
    FROM answer_records ar 
    WHERE ar.question_id = w.question_id 
      AND ar.user_id = w.user_id 
      AND ar.is_correct = 0
    ORDER BY ar.answered_at DESC 
    LIMIT 1
) as user_answer
```

**测试结果：**
- ✅ 修复前：返回19条记录（有重复）
- ✅ 修复后：返回10条记录（无重复）

## 如何排查"错题不显示"的问题

### 步骤1：打开浏览器控制台

1. 在错题本页面按 `F12` 打开开发者工具
2. 切换到 **Console（控制台）** 标签

### 步骤2：查看日志输出

刷新错题本页面，应该看到以下日志：

```
开始加载错题本，用户ID: 148
API返回数据: {count: 10, wrong_book: Array(10)}
错题数据已加载，数量: 10
渲染错题本，当前错题数量: 10
筛选条件: all
```

### 步骤3：根据日志判断问题

#### 情况A：看到"用户未登录，无法加载错题本"
**原因：** 用户没有登录
**解决：** 先登录账号

#### 情况B：看到"API返回数据: {count: 0, wrong_book: []}"
**原因：** 该用户确实没有错题
**解决：** 去答题页面故意答错几道题，然后刷新错题本

#### 情况C：看到"未找到 wrong-book-list 元素"
**原因：** HTML结构有问题
**解决：** 检查wrong-book.html中是否有 `<div id="wrong-book-list">`

#### 情况D：看到"没有错题数据，显示空状态"
**原因：** appState.wrongBook为空数组
**解决：** 检查API是否正常返回数据

#### 情况E：没有任何日志输出
**原因：** loadWrongBook()函数没有被调用
**解决：** 检查wrong-book.html的window.onload是否正确

## 常见问题和解决方案

### 问题1：页面显示"暂无错题"但实际有错题

**可能原因：**
1. 用户ID不正确
2. API请求失败
3. 数据没有正确加载到appState

**排查方法：**
```javascript
// 在控制台执行
console.log('用户ID:', appState.userId);
console.log('错题数据:', appState.wrongBook);
```

**解决方法：**
- 确认已登录
- 点击"🔄 刷新错题"按钮
- 检查网络请求是否成功

### 问题2：能看到题目但没有显示用户答案

**可能原因：**
- 数据库中answer_records表没有该用户的错误答题记录

**排查方法：**
```python
# 在后端执行
import sqlite3
conn = sqlite3.connect('quiz.db')
cursor = conn.cursor()
cursor.execute('''
    SELECT user_id, question_id, user_answer, is_correct 
    FROM answer_records 
    WHERE user_id = ? AND is_correct = 0
    LIMIT 5
''', (user_id,))
print(cursor.fetchall())
```

**解决方法：**
- 确保答题时正确调用了submit_answer API
- 检查is_correct参数是否正确传递

### 问题3：筛选功能不工作

**可能原因：**
- initWrongBookFilters()没有被调用
- 筛选按钮的data-subject属性不正确

**排查方法：**
```javascript
// 在控制台执行
const buttons = document.querySelectorAll('.filter-btn');
buttons.forEach(btn => {
    console.log(btn.dataset.subject, btn.textContent);
});
```

**解决方法：**
- 确认HTML中有正确的data-subject属性
- 重新加载页面
