# 错题本实时更新功能说明

## 问题描述
之前错题本页面加载后不会自动更新数据，用户答题后需要手动刷新浏览器才能看到新的错题。

## 解决方案

### 1. 手动刷新按钮
在错题本页面顶部添加了"🔄 刷新错题"按钮，用户可以随时点击刷新最新数据。

**功能特点：**
- 点击后显示"⏳ 刷新中..."状态
- 刷新成功后显示"✅ 已刷新"提示（1.5秒）
- 刷新失败显示"❌ 刷新失败"提示（2秒）
- 保持当前的筛选条件不变

**位置：** [wrong-book.html](file://c:/Users/j2846/Desktop/my%20attempt/flask_app/static/wrong-book.html#L81-L83)

### 2. 页面可见性自动刷新
当用户从其他标签页或应用切换回错题本页面时，自动刷新数据。

**实现方式：**
```javascript
// 监听页面可见性变化
document.addEventListener('visibilitychange', function() {
    if (!document.hidden && appState.userId) {
        loadWrongBook();
    }
});

// 监听窗口获得焦点事件
window.addEventListener('focus', function() {
    if (appState.userId) {
        loadWrongBook();
    }
});
```

**触发场景：**
- 从其他浏览器标签页切换回来
- 从其他应用程序切换回来
- 最小化后恢复窗口

**位置：** [wrong-book.html](file://c:/Users/j2846/Desktop/my%20attempt/flask_app/static/wrong-book.html#L115-L129)

### 3. 答题时自动刷新
用户在答题页面答错题目时，会自动刷新错题本数据到内存中。

**实现位置：** [app.js](file://c:/Users/j2846/Desktop/my%20attempt/flask_app/static/app.js#L789-L790)
```javascript
// 刷新错题本
await loadWrongBook();
```

**注意：** 这只会更新内存中的数据，如果用户已经在错题本页面，需要手动刷新或等待自动刷新。

## 技术细节

### 后端API
- **接口：** `/api/user/<user_id>/wrong-book`
- **方法：** GET
- **返回：** 最新的错题列表（包含用户答案和正确答案）

### 前端函数

#### `loadWrongBook()`
- 从后端API获取最新错题数据
- 更新 `appState.wrongBook` 状态
- 调用 `renderWrongBook()` 渲染页面
- 调用 `initWrongBookFilters()` 初始化筛选器

#### `renderWrongBook(filterSubject)`
- 根据筛选条件过滤错题
- 动态生成HTML并插入页面
- 支持按科目筛选（全部、数学、语文、英语、科学、历史）

#### `initWrongBookFilters()`
- 为筛选按钮添加点击事件
- 为刷新按钮添加点击事件
- 处理刷新状态和错误提示

## 使用建议

### 推荐操作流程：
1. **答题过程中：** 答错的题目会自动加入错题本
2. **查看错题：** 切换到错题本页面，数据会自动刷新
3. **继续答题：** 再次答错后，错题本的错误次数会增加
4. **返回查看：** 切换回错题本页面，会看到更新后的数据

### 注意事项：
- 如果数据没有及时更新，可以手动点击"刷新错题"按钮
- 确保用户已登录（`appState.userId` 有值）
- 网络异常时刷新会失败，会有错误提示

## 测试方法

1. **测试手动刷新：**
   - 打开错题本页面
   - 去答题页面答错几道题
   - 返回错题本页面，点击"刷新错题"按钮
   - 应该能看到新增的错题

2. **测试自动刷新：**
   - 打开错题本页面
   - 切换到其他标签页
   - 去答题页面答错几道题
   - 切换回错题本标签页
   - 应该能自动看到更新的错题

3. **测试筛选功能：**
   - 刷新后，点击不同的科目筛选按钮
   - 确认只显示对应科目的错题
